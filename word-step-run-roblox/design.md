# Word Step Run Design

## One-Line Pitch
固定コマンドを押すと単語が実体化し、その文字を足場にして空中コースを走る対戦アクションレース。

## Why This Works
- 一目で伝わる
  - `文字が足場になる` はサムネだけでも理解しやすい
- 動画映えする
  - 空中に `STEP` や `BRIDGE` が連続で出る
- 友達と遊ぶ理由が強い
  - 自分の足場を横取りされたり、同じ足場に集まって事故る
- モバイル対応しやすい
  - 音声入力なし、固定ボタンだけで成立する

## Product Principles
- 操作は3ボタンまでに絞る
- 最初の10秒で最初のギャップを越えさせる
- 自分の足場が他人にも影響するようにする
- コマンドは読む必要がなく、色と形で使い分けられるようにする

## Fantasy
本物のマイクは使わない。
体験としては `言葉を固めて道を作る` が本質。
プレイヤーはコマンドボタンを押して、空中に短時間だけ残る 3D ワードブロックを出す。

## MVP Game Loop
1. Lobby
2. 3秒カウントダウン
3. 75秒レース
4. 上位3人表示
5. 次ラウンドへ

## MVP Commands
### `STEP`
- 用途: 目の前に 1 枚の足場を出す
- 役割: 基本移動
- クールダウン: 短い
- 寿命: 短い

### `BRIDGE`
- 用途: 前方に 3 枚ぶんの長い足場を出す
- 役割: 大きいギャップを越える
- クールダウン: 中
- 寿命: `STEP` より少し長い

### `UP`
- 用途: 1段上の足場を出す
- 役割: 段差や高低差対応
- クールダウン: 中
- 寿命: 短い

## Block Rules
- すべてのブロックは生成者の前方に出る
- ブロックは 2.5 - 4.0 秒で消える
- 他プレイヤーも乗れる
- 同じ位置に重ねて出せない
- 消える 0.4 秒前に点滅する

## Course Structure
MVP は 1 マップだけでいい。
ただし 30 秒の中に3種類の要求を必ず入れる。

### Section A: Learning
- 小さいギャップだけ
- `STEP` を押せば進める

### Section B: Distance Check
- 長めの穴
- `BRIDGE` を使わないと落ちる

### Section C: Height Check
- 上下の段差
- `UP` を覚えさせる

### Section D: Chaos Finish
- 足場が狭く、人が密集する
- 他人の足場に便乗するのが強い

## Player Interaction
MVP では PvP ダメージを入れない。
その代わり、以下だけ残す。

- 体当たりで軽く押し出せる
- 他人のブロックに乗れる
- 同じ場所に密集すると落ちやすい

このくらいが一番軽くて、事故で笑いが起きる。

## Camera / Feel
- 三人称固定寄り
- FOV は少し広め
- 生成した文字ブロックはポップに飛び出す
- 着地時に軽い squash / bounce を入れる

## Visual Direction
- 空中コース
- 文字ブロックは太く読みやすい
- `STEP` は白、`BRIDGE` は黄、`UP` はシアン
- 背景は明るめの空で、足場シルエットを見やすくする

## UI
- 下部中央に 3 つのコマンドボタン
- 各ボタンに残りクールダウン表示
- 上部に順位と残り時間
- 初回 15 秒だけ操作ヒントを表示

## Win Condition
- 先にゴールした順で順位確定
- タイムオーバー時は進行距離で順位確定

## Why Not Use Real Voice
- マイク必須にすると母数が狭くなる
- 音声品質で事故る
- モバイルや子どもユーザーの導線が悪くなる

固定コマンドにすると、体験の核だけ残して広く配れる。

## MVP Scope
- 1マップ
- 3コマンド
- 1ゲームモード
- 8人対戦
- シンプルな順位 UI

削るべきもの:
- 課金アイテム
- 複雑な成長要素
- 複数マップ
- ランク制
- 本物の音声入力

## Phase 2
- `PUSH` コマンド追加
- 崩れる足場
- 日替わりコース
- 2v2 協力モード
- 見た目だけ変わるワードスキン

## First Prototype Goal
最初の試作で確認したいのは1つだけ。

`空中でコマンドを押して、自分の出した文字を踏んで進む` のが気持ちいいか。

ここが弱ければ、それ以上の機能は意味がない。
