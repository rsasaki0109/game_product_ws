# Blink Door Brawl Design

## One-Line Pitch
入口ドアと出口ドアを置いて、相手の進行ルートや逃げ道を書き換えながらリングアウトを狙う近距離 PvP。

## Why This Works
- 見た瞬間にルールが伝わる
- 事故と裏取りが自然に起きる
- 友達同士の煽り合いと笑いが出やすい
- 既存シューターほどエイム依存ではない

## Core Fantasy
ドラえもん要素は使わない。
体験の核は `どこでもつながるドアで空間を壊す` こと。

## MVP Loop
1. ロビー
2. 3秒カウントダウン
3. 90秒バトル
4. リザルト
5. 次ラウンド

## Actions
### `Q`: Entry Door
- 自分の入口ドアを前方床面に設置
- 既存入口は上書き

### `E`: Exit Door
- 自分の出口ドアを前方床面に設置
- 既存出口は上書き

### `F`: Push
- 前方の相手を短く押し出す
- ドアへ押し込む、奈落へ落とすための補助

## Door Rules
- 入口に触れると出口へ即ワープ
- 出口が未設置なら入口は無効
- 入口は全員が使える
- 入口を通った直後は短時間だけ再ワープしない
- 相手を自分のドア経由で飛ばした直後は、一定時間 `自分が落とした可能性がある状態` として記録する

## Kill Rule
- プレイヤーが奈落へ落ちたら直近の脅威元にキル付与
- 脅威元は以下
  - 相手のドアでワープさせられた
  - 相手のプッシュを受けた

## Arena Direction
- 中央プラットフォーム + 周囲4島
- 全面に穴が多い
- 柵は置かない
- 外周に逃げ場はあるが安全地帯はない

## MVP Scope
- FFA のみ
- 武器なし
- 成長要素なし
- 1マップ固定
- 入口 / 出口 / プッシュの3アクションのみ

## Phase 2
- ドアの寿命差分
- 可動床
- 崩れる島
- 2v2 モード
- ドア見た目スキン

## First Prototype Goal
最初に見るべきなのは1点だけ。

`入口と出口を置いて、相手をドアか奈落へ押し込む` だけで笑えるか。
